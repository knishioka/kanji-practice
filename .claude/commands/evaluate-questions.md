# 問題評価コマンド

生成される問題の品質を評価し、教育的観点からレビューします。

## 評価項目

### 1. 問題生成の網羅性
- 指定学年の全漢字から問題が生成できるか
- 各モードで十分な問題数が生成できるか
- データ不足による問題生成不可がないか

### 2. 問題の難易度バランス
- 学年相応の難度か
- 読みの難易度（音読み/訓読み）
- 例語の適切性（小学生が理解できる語彙か）

### 3. 学習効果の評価
- 分散学習: 同じ漢字が適度に繰り返されるか
- 間隔反復: ランダム出題で効果的か
- 学習順序: strokeCount→reading→writing→sentence

### 4. モード別評価
| モード | 評価観点 |
|--------|----------|
| reading | 読みが一意に特定できるか |
| writing | 書き取りの手がかりが適切か |
| strokeCount | 画数情報が正確か |
| sentence | 例文が自然か |
| homophone | 同音異義語のグループが適切か |
| radical | 部首情報が正確か |
| okurigana | 送りがなデータが正確か |
| antonym | 対義語/類義語が同学年内か |

### 5. 統計レポート
- 各学年の漢字数
- モード別の問題生成可能数
- データカバレッジ率

## 使用方法

```
/evaluate-questions <学年>
/evaluate-questions --all
/evaluate-questions --mode <モード>
/evaluate-questions --report  # 統計レポート出力
```

## 実行手順

1. `npm run test:unit -- --grep "問題生成データ検証"` を実行
2. テスト出力から統計情報を確認
3. 問題のある箇所を特定してレポート
4. 改善提案を提示

$ARGUMENTS で学年またはモードを指定してください。
